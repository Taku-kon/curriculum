■ 開発
ウォーターフォール開発
 開発手順を一つずつ確認しながら工程を進めていく手法。失敗すると先に進めない。先に進むと戻れない。
アジャイル開発
 クライアントの要望に応えるシステムをできる限り早くリリースする開発手法。開発メンバーがシステムにおける優先度を順位を付け、短い期間での納品を目指して動く。
マイグレーション
ソフトウェアのシステムを全面的に刷新して新しい環境に置き換えること。既存システムのどの部分を残すかによって様々な手法が用いられる。
デグレーション
デグレードのこと。マイグレやアップグレードした事によってバグなどが発生し悪化してしまうことを指す
デプロイ
開発したソフトウェアを実際の運用環境に展開して実用することを指す
コンフリクト
プログラムなどが強豪状態に陥ってしまうこと
環境
ソフトウェアなどからみたコンピューター全体のこと
開発環境（ローカル）
開発環境はシステムを開発するために必要な環境のこと。ローカルはネット上に接続していない狭い範囲のこと。
検証環境（ステージング）
実際の本番環境に似せて作った動作確認用のテスト環境のこと
本番環境
実際にシステムが稼働している環境のこと
設計書／仕様書
工業製品やソフトウェア、情報システム、サービスなどを開発する際に、備えるべき機能やその性能、特性、満たすべき要件などを図表や文章などで記述した文書
要件定義
開発やプロジェクトを始める前の檀家で必要な機能や要求を分かりやすくまとめていく作業のこと
基本設計書
ソフトウェアの動きを外から見たときにどういう動作をしているかを決めたもの。（what）
詳細設計書
基本設計で決められた動きをどうやって実現するのかを決めたもの。（how）
テスト
単体テスト（UT）
一つの関数、モジュールが正常に作動するかのテスト
結合テスト（CT／IT）
複数の関数、モジュールを組み合わせて、正常に作動するかのテスト
総合テスト（ST）
実際の運用携帯に沿ってテストする。
エビデンス／証跡
システム開発のテスト工程などで、システムが正しく動作している、あるいは不具合が生じたことを示す証拠となる書類やデータなど
データパッチ
プログラムやデータを更新・修正するために、新しい要素との差分を抜き出したデータのこと
Access
機器やソフトウェア、システム、情報などを利用（操作、取得、閲覧、読み書きなど）可能な状態にすること
マクロ
関連する複数の操作や手順、命令などを一つにまとめ、必用に応じて呼び出すことが出来るようにする機能のこと
VBA
Visual Basic for Applicationsの略。Microsoft Officeに含まれるアプリケーションソフトの拡張機能で、利用者が簡易なプログラムを記述して実行することで複雑な処理の自動化などを行なうことができるもの
GAS（Google Apps Script）
Googleが開発・提供しているプログラミング言語
API
ソフトウェアやプログラム同士を繋げるインターフェース
WEB API
web上でAPIのやりとりを可能にしたもの
REST API
RESTの４原則に則ったAPI（統一インターフェース、アドレス可能性、接続性、ステートレス性）
SOAP
異なるコンピュータ上で動作するプログラム同士がネットワークを通じてメッセージを伝え合い、連携して動作するための通信プロトコルの一つ
Ajax
非同期通信をjavascriptの技術で実行すること
XML
拡張可能なマークアップ言語。W3Cによるオープン規格。
JSON
javascriptのオブジェクト記法を用いたデータ交換フォーマット。JSONを間に挟むことで他のプログラミング言語間のデータの受け渡しが楽になる
ステータスコード
システムが処理結果や現在の状態を外部に知らせるために発する数字や短い符号。
クロスオリジン（Cross-Origin）／CORS（Cross-Origin Resource Sharing）
Webサーバに特殊なHTTPヘッダ項目を追加することで、送り出したWebページ上のスクリプトがWebブラウザから別のサーバへアクセスできるようにすること
バリデーション
入力内容や記述内容が要件を満たしているか、妥当性をチェックすること
データ連携
Daemon
UNIX系のOSでメインメモリ上に常駐して特定の機能を提供するプログラム。
CRON
Linuxサーバーの管理で大切な設定方法の一つ。
バッチ
一回分にまとめるという意味。大量のデータを一括処理する際の処理の単位をバッチと呼ぶ。
ログ
コンピュータの利用状況やデータ通信など履歴や情報の記録を取る事、またその記録を指す。
CSV
データ形式の一つ項目をカンマで区切って列挙したもの。
TSV
複数の項目で構成されるデータを複数件列挙して表現することができる汎用的なデータ形式の一つ
プルリクエスト（マージリクエスト）
コードなどを追加・修正した際、本体への反映を他の開発者に依頼する機能。

株式会社ヌーラボが提供するプロジェクト管理ツール
Jooto

Trello

■ バージョン管理
集中型
ネットワークに接続した状態で中央のリポジトリを複数人で共有
分散型
各自、個人のパソコン上にリポジトリを持ち、好きなタイミングで好きなリポジトリに同期する
Github
管理するデータをインターネット上で共有、公開することができるネットワークサービスの一つ。
Backlog
ASP型プロジェクト管理ツール
SourceTree
Gitを視覚的に操作できるGUIツール
Tortoise SVN
バージョン管理ツール
Git Flow
Gitの機能であるブランチを活用したGitの開発手法でもあり、ツールの名前でもある。運用モデル
 master
  デフォルトのブランチ。元のデータになるのでmasterで作業は基本的にしない。
 release
  リリース作業を行うためのブランチ
 develop
 開発の主軸になるブランチ
 feature
 機能追加、作業修正を行うためのブランチ
■ FW／ライブラリ
FW
アプリケーションを開発するとき、その土台として帰納させるソフトウェアのこと
Spring Framework
javaの開発環境で使用できるオープンソースのフレームワーク
Struts
Java Servlet APIを拡張してMVC (Model、View、Controller)アーキテクチャを採用した、オープン・ソースのフレームワークです。JSPページ、JavaBeans、リソース・バンドル、XMLなどの標準テクノロジに基づいた、管理可能で拡張可能な柔軟性のあるWebアプリケーション
JSF（Java Server Faces）
Java Webアプリケーション用のユーザー・インタフェース(UI)フレームワーク
ライブラリ
便利なプログラムのコードが集められたファイルのこと
Bootstrap
フロントエンドライブラリ、Twitter社が開発したCSSライブラリ
Flyway
オープンソースのデータベースマイグレーションツール
Jackson
JSON形式のデータを処理するための高速かつ軽量なJavaライブラリ
jQuery
JavaScriptのためのライブラリ。コード量の大幅な短縮と、異なるブラウザでも同じコードで処理が行なえる点
Junit
ava言語で開発されたプログラムの単体テストを行なうためのソフトウェア
Log4j
Javaがベースのデバイスにおいて、ログの保存や出力に使われるプログラム
Lombok
Java特有の冗長なコードを簡潔にしてくれるオープンソースのライブラリ
ORM
オブジェクト関連マッピング。データベースとオブジェクト指向プログラミング言語の間の非互換なデータを変換するプログラミング技法
■ 連絡ツール

Slack

IP Messenger
LAN用のメッセンジャーソフト

アノテーション
特定のデータに情報タグを付加する。注記、注釈のことなど
メモリ管理

 ユーザーが作ったプログラムが利用するメモリ領域のこと
 GC
 ガベージコレクション。機能の一つで、実行中のプログラムが占有していたメモリ領域のうち不要になったものを自動的に解放し、空き領域として再利用できるようにするもの
GoF

デザインパターン
ソフトウェア開発の生産性や保守性を上げることを目的に過去に編み出した設計ノウハウを蓄積しカタログ化したもの
■ その他
オブジェクト指向
システム構成の考え方の事
バッチ処理
まとめて行う処理のこと。自動化できるメリット。忙しくないときに実行できるメリットなどがある
コンパイル
プログラミング言語で書かれたコンピュータプログラムを解析し、コンピュータが直接実行可能な形式のプログラムに変換すること
カバレッジ
対象範囲に対して全体の内どれくらい網羅しているかを示す指標。ソフトウェア開発の領域ではテストの実施状況をモニタリングする方法を示す言葉。論理構造全体に対してどれだけの実行ができたかを網羅性の尺度
プラットホーム
サービスやシステム、ソフトウェアを提供・カスタマイズ・運営するために必要な共通の土台（基盤）となる標準環境
